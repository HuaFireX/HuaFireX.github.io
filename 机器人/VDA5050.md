
# VDA5050 协议对接开发指南

> VDA5050快速上手

---

## 1. 概述

VDA5050 是 AGV 与主控系统之间的通信协议，基于 MQTT + JSON。

**协议目标**：实现不同厂商 AGV 的混合调度

**当前 RCS4.3 状态**：使用自定义 TCP 协议，需要改造为 VDA5050

---

## 2. MQTT 主题结构

```
{interfaceName}/{majorVersion}/{manufacturer}/{serialNumber}/{topic}

例：uagv/v2/YourCompany/AGV001/order
```

### 六大主题

| 主题               | 方向        | 说明     | 必选 |
| ------------------ | ----------- | -------- | ---- |
| `order`          | 主控 → AGV | 任务命令 | ✅   |
| `instantActions` | 主控 → AGV | 即时动作 | ✅   |
| `state`          | AGV → 主控 | 状态上报 | ✅   |
| `factsheet`      | AGV → 主控 | AGV能力  | ✅   |
| `connection`     | AGV → 主控 | 连接状态 | ✅   |
| `visualization`  | AGV → 主控 | 可视化   | ❌   |

## 3. 核心概念

### 3.1 Base vs Horizon

```
released=true  → Base（已确认，必须走）
released=false → Horizon（待确认，可能变化）

Example:
nodes: [
  {nodeId: "n1", released: true},   // Base
  {nodeId: "n2", released: true},   // Base  
  {nodeId: "n3", released: false},  // Horizon
  {nodeId: "n4", released: false}   // Horizon
]

AGV 只需要走到 n2，然后等主控发新的 orderUpdate
```

### 3.2 orderUpdate（增量更新）

- 保持 `orderId` 不变
- `orderUpdateId` 递增（0→1→2...）
- 第一个节点 = 上一个base的最后一个节点（接续点）

```
场景：主控发现前方路况变化，需要调整路线
  orderUpdateId=0 ──► orderUpdateId=1 ──► orderUpdateId=2
  [n1,n2,n3○]       [n2,n4,n5○]         [n4,n6,n7○]
                       ↑
                   从n2继续（接续点）
```

## 4. 核心消息格式

### 4.1 factsheet（AGV能力上报）

**发送时机**：AGV 连接时发送，`retain=True`

```python
factsheet = {
    "manufacturer": "YourCompany",
    "serialNumber": "AGV001",
    "agvClass": "FORKLIFT",          # FORKLIFT / CONVEYOR / TOWTRACTOR
    "maxLoadMass": 1000,
    "maxSpeed": 1.5,
    "maxHeight": 2.5,
    "supportedActions": [
        "startPause", "stopPause",
        "cancelOrder", "startCharging", "stopCharging",
        "pick", "drop", "initPosition"
    ],
    "protocolFeatures": ["order", "instantActions", "state", "factsheet", "connection"]
}
```

### 4.2 order（任务命令）

**发送方**：主控 → AGV

```json
{
  "headerId": 1,
  "timestamp": "2024-04-01T10:00:00.00Z",
  "version": "2.0.0",
  "manufacturer": "YourCompany",
  "serialNumber": "AGV001",
  "orderId": "order_001",
  "orderUpdateId": 0,
  "nodes": [
    {
      "nodeId": "node1",
      "sequenceId": 0,
      "released": true,
      "nodePosition": {"x": 1.0, "y": 2.0, "theta": 0.0, "mapId": "map_01"},
      "actions": []
    }
  ],
  "edges": [
    {"edgeId": "e1", "sequenceId": 1, "released": true, "startNodeId": "node1", "endNodeId": "node2"}
  ]
}
```

**关键字段说明**：

| 字段               | 说明                  |
| ------------------ | --------------------- |
| `orderId`        | 任务唯一ID            |
| `orderUpdateId`  | 任务版本号（递增）    |
| `released=true`  | 已确认路径（base）    |
| `released=false` | 待确认路径（horizon） |
| `sequenceId`     | 节点/边的顺序号       |

### 4.3 instantActions（即时动作）

**发送方**：主控 → AGV

```json
{
  "headerId": 2,
  "timestamp": "2024-04-01T10:05:00.00Z",
  "version": "2.0.0",
  "manufacturer": "YourCompany",
  "serialNumber": "AGV001",
  "actions": [
    {"actionType": "startPause", "actionId": "action_001"}
  ]
}
```

**常用动作类型**：

| actionType        | 说明             |
| ----------------- | ---------------- |
| `startPause`    | 暂停             |
| `stopPause`     | 恢复             |
| `cancelOrder`   | 取消订单         |
| `stateRequest`  | 请求立即上报状态 |
| `startCharging` | 开始充电         |
| `stopCharging`  | 停止充电         |
| `pick`          | 拾取             |
| `drop`          | 放置             |
| `initPosition`  | 初始化位置       |

### 4.4 Action 参数详解

**pick（取货）**：

```json
{
  "actionType": "pick",
  "actionId": "pick_001",
  "actionParameters": {
    "loadId": "托盘001",
    "loadType": "EPAL",
    "height": 100
  }
}
```

**drop（放货）**：

```json
{
  "actionType": "drop",
  "actionId": "drop_001",
  "actionParameters": {
    "loadId": "托盘001",
    "loadType": "EPAL",
    "height": 100
  }
}
```

**initPosition（定位初始化）**：

```json
{
  "actionType": "initPosition",
  "actionId": "init_001",
  "actionParameters": {
    "x": 1.5,
    "y": 2.3,
    "theta": 0.785,
    "mapId": "map_01"
  }
}
```

### 4.5 state（状态上报）

**发送方**：AGV → 主控

```json
{
  "timestamp": "2024-04-01T10:10:00.00Z",
  "version": "2.0.0",
  "manufacturer": "YourCompany",
  "serialNumber": "AGV001",
  "orderId": "order_001",
  "orderUpdateId": 0,
  "lastNodeId": "node1",
  "nodeStates": [
    {"nodeId": "node1", "sequenceId": 1, "nodeState": "VISITED", "timestamp": "..."}
  ],
  "position": {"x": 1.0, "y": 2.0, "theta": 0.0, "mapId": "map_01"},
  "velocity": {"x": 0.0, "y": 0.0, "theta": 0.0},
  "batteryState": 95,
  "charging": false,
  "loadState": "EMPTY",
  "paused": false,
  "errors": []
}
```

**nodeState 状态流转**：

```
TRAVERSING（行驶中） → VISITED（已到达） → FINISHED（完成）
```

### 4.6 connection（连接状态）

```json
{"state": "online"}
{"state": "offline"}
```

### 5.3 order 解析示例

```python
def handle_order(order_data):
    order_id = order_data["orderId"]
    nodes = order_data["nodes"]
  
    # 提取已释放的路径（base）
    base_nodes = [n for n in nodes if n.get("released", False)]
  
    # 转换为内部任务格式
    path_points = []
    for node in base_nodes:
        pos = node.get("nodePosition", {})
        path_points.append((pos["x"], pos["y"], pos.get("theta", 0)))
  
    execute_path(path_points)
```

### 基础类位置


后续迁移到RCS源码时，放到 `layer1_communication/vda5050/`

### 核心流程

1. 连接 MQTT broker
2. 发布 factsheet（retain=True）
3. 发布 connection: online
4. 订阅 order 和 instantActions
5. 收到 order → 解析执行 → 上报 state
6. 收到 instantActions → 响应 → 上报 state
